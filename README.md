# Финальный проект 7 спринта: API‑тесты Яндекс Самокат

Проект содержит автотесты для API учебного сервиса **Яндекс Самокат**.

Тестируемое API: `https://qa-scooter.praktikum-services.ru/api/v1`

## Стек

- Python 3
- pytest
- requests
- allure-pytest

## Структура проекта

```text
sprint_7/
  conftest.py                 # фикстуры (создание/удаление курьера)
  requirements.txt            # зависимости проекта
  src/
    api/
      courier_api.py          # обёртки над API курьеров
      orders_api.py           # обёртки над API заказов
    utils/
      generators.py           # генераторы тестовых данных (логины и т.п.)
    data/
      messages.py             # константы с текстами сообщений API
  tests/
    test_courier_create.py        # тесты ручки "Создать курьера"
    test_courier_login.py         # тесты ручки "Логин курьера"
    test_courier_delete.py        # тесты ручки "Удаление курьера"
    test_orders_create.py         # тесты ручки "Создать заказ"
    test_orders_get_by_track.py   # тесты ручки "Получить заказ по треку"
    test_orders_accept.py         # тесты ручки "Принять заказ"
    test_orders_list.py           # тесты ручки "Список заказов"
```

## Установка и запуск

1. Клонировать репозиторий и перейти в папку проекта:

```bash
git clone <URL_ВАШЕГО_РЕПОЗИТОРИЯ>
cd sprint_7
```

2. Создать и активировать виртуальное окружение (Windows, PowerShell):

```bash
python -m venv venv
.\venv\Scripts\Activate.ps1
```

Для Linux / macOS:

```bash
python3 -m venv venv
source venv/bin/activate
```

3. Установить зависимости:

```bash
python -m pip install -r requirements.txt
```

4. Запустить все тесты:

```bash
pytest -v
```

5. Запуск тестов с сохранением результатов для Allure:

```bash
pytest -v --alluredir=allure_results
```

Если установлен Allure CLI, для просмотра отчёта выполните:

```bash
allure serve allure_results
```

## Реализованные проверки

### Курьеры

- Создание курьера:
  - успешное создание с валидными данными;
  - создание курьера с уже существующим логином;
  - создание без логина;
  - создание без пароля.

- Логин курьера:
  - успешный логин существующего курьера;
  - логин с неверным паролем;
  - логин с несуществующим логином;
  - логин без логина;
  - логин без пароля.

- Удаление курьера:
  - успешное удаление существующего курьера;
  - попытка удалить курьера с несуществующим `id`.

### Заказы

- Создание заказа:
  - успешное создание с разными вариантами поля `color`
    (`["BLACK"]`, `["GREY"]`, `["BLACK", "GREY"]`, `[]`).

- Получение заказа по треку:
  - успешное получение заказа по существующему `track`;
  - попытка получить заказ по несуществующему `track`;
  - попытка запроса без указания `track`.

- Принятие заказа курьером:
  - успешное принятие заказа существующим курьером;
  - попытка принять несуществующий заказ;
  - попытка принять заказ несуществующим курьером;
  - попытка принять заказ без указания `courierId`.

- Получение списка заказов:
  - успешное получение списка, проверка кода ответа и структуры тела.
